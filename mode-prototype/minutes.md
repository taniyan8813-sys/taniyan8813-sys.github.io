# 議事録

## 2026-05-13

### 目的
- 2つのゲーム構造を持つブラウザゲームのプロトタイプを作る。
- 世界観、ストーリー、キャラクター設定は仮置き。
- 主目的は、Mode A と Mode B のゲーム構造を同じ土台で比較できるようにすること。

### 実装内容
- `index.html`, `styles.css`, `game.js` で依存なしのブラウザゲームを作成。
- Mode A は拠点防衛寄りの構造として実装。
  - 中央拠点を敵が狙う。
  - プレイヤーはキャンバスクリックで防衛火力を追加できる。
  - ウェーブ進行に応じて敵の圧力が上がる。
- Mode B は Vampire Survivors 寄りの構造として実装。
  - プレイヤーキャラが前進し続ける。
  - 敵の群れが接近する。
  - プレイヤーは移動し、自動攻撃で敵を倒す。
- 画面右側に Mode, Wave, Integrity, Score を表示。
- Reset と Pause を追加。

### 判断
- 今回は構造実験が目的なので、アセット制作や世界観設定には踏み込まない。
- フレームワークや外部ライブラリは使わず、静的ファイルだけで動く形にした。
- 2つのモードは同じステータス表示とゲームループを共有し、比較しやすくした。

### 未決事項
- Mode A のタワー配置コスト、射程、種類。
- Mode B の成長要素、武器選択、経験値。
- 2モードを別々のゲームとして分けるか、1つのキャンペーン内で接続するか。
- 今後の仮世界観とキャラクター設定。

## 2026-05-13 追記

### ユーザー要望
- タワー配置で通路を塞ぐ一般的なタワーディフェンスではなく、固定された自陣・拠点・キャラクター側から敵の群れを迎撃する防衛ゲームに寄せる。
- イメージはライン防衛、シューティング、ローグライト成長の組み合わせ。
- 今は正式な世界観やキャラクター設定を入れず、防衛ゲームとして成立する骨組みを優先する。

### 変更内容
- 2モード切り替え構成をやめ、`Line Defense` という防衛特化プロトタイプへ変更。
- 守る対象を画面下部に固定配置。
- 敵は画面上・左右から出現し、守る対象へ向かって進む。
- 敵が守る対象に到達する、または防衛ラインを越えると Base HP が減る。
- Base HP が 0 になるとゲームオーバー。
- プレイヤー側は固定位置から自動射撃する。
- クリックまたはドラッグで攻撃方向を指定できる。
- 敵を倒すと XP と Score が入る。
- XP が一定値に達するとゲームが一時停止し、3択の簡易アップグレードを選べる。
- 一定時間生き残ると次ウェーブへ進み、Wave 5 を越えるとクリア扱い。
- 確認環境でもゲームループが進むように、描画更新は固定タイマー駆動にした。

### 実装したアップグレード
- Rapid Fire: 攻撃間隔を短縮。
- Heavy Rounds: 弾のダメージを増加。
- Side Barrel: 斜め方向の追加弾を増やす。
- Field Repair: Base HP を回復。
- Anchor Field: 敵の速度を少し下げる。

### 次に検証したいこと
- 手動照準と自動照準のどちらがこのゲーム構造に合うか。
- 敵の出現方向を「画面奥からの圧」として見せる方法。
- ウェーブ終了時だけ強化するか、敵撃破によるレベルアップ時に強化するか。
- スキルを通常攻撃とは別の意思決定として入れるか。

## 2026-05-14 追記

### レビュー結果と修正
- タッチ操作がキャンセルされた場合に照準ドラッグ状態が残る可能性があったため、`pointercancel` で解除するよう修正。

### 追加内容
- EAEが定期的にログ介入する演出を追加。
- 介入時に画面へ短い紫系フラッシュと走査線を出すようにした。
- RESTOREが進むと、EAEログの文言が少し不穏になるようにした。
- ゲーム内表示文言を日本語化した。
- EAE介入時に短時間だけ射撃間隔が短くなるようにし、ログ介入がゲーム挙動にも影響するようにした。

### 意図
- EAEはまだ明確な敵として説明しない。
- ただし、情報源を独占している違和感を画面上のログで少し出す。
- 日本語表示にして、試作版として遊ぶ時の読み取り負荷を下げる。
